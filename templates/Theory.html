Algorithm properties

Before going into the specifics of the flowchart rules it is good to define the essential characteristics that each algorithm should have. An algorithm to be defined as such must have 6 essential characteristics:
1) Finiteness: each algorithm must be finite or better it must have a limited sequence of instructions;
2) Executable: each instruction must be executed according to the characteristics of the performer. Or rather, any algorithm we are going to realize must be able to be run on any computer. The computer must be able to interpret and execute instructions;
3) Unambiguous: each instruction must be interpreted univocally by the executor. The algorithm cannot be subjective but objective;
4) General: it must be valid for a class of problems;
5) Deterministic: starting from the same initial data it must always obtain the same result regardless of the performer;
6) Complete: I have to cover all the cases that can occur during the execution.

Introduction to algorithms

An algorithm is nothing more than a simple procedure that allows the solving a class of problems by applying a sequence finished with precise instructions which, in turn, must be interpreted and carried out until their conclusion following a very specific order. The idea of algorithm is central to computer science. Some algorithms are faster than others and most of the algorithms that have been discovered have made it possible to solve problems that previously took an incredible amount of time. (example, finding the best way to stow packages in a container, or find if very large numbers (100-digits and more) are prime.)
